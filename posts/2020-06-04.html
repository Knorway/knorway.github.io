Next.js useRouter의 router.query는 undefined를 뱉을 수 있다

2020-06-04 ·

Next
const { uuid } = router.query
const { data } = useSWR(
        `${API_URL}/users/${uuid}`,
        async (url) => {
            const response = await client.get(url, { headers: BearerHeader() });
            return response.data;
        },
        { revalidateOnFocus: false }
    );

이 단순한 fetcher는 가끔 /users/undefined로 404가 뜰 수 있다. 정적으로 최적화 된 페이지는 경로 파라미터 없이 제공되고 Hydrate 이후 Next.js가 query 객체를 채우게 되기 때문이다.

이를 해결하는 가장 일반적인 방법은 아래와 같을 것이고

useEffect(()=>{
    if(!router.isReady) return;
    // router.query를 사용하는 로직


}, [router.isReady]);

위와 같이 useSWR을 사용하는 경우는

if (!uuid) return;
const response = await client.get(url, { headers: BearerHeader() });

이렇게 한 줄 더 넣어주자. return; 은 undefined를 리턴하게 되고 useSWR은 넘겨진 콜백함수의 페처가 유의미한 값, 즉 데이터나 프로미스 오류를 반환하게 되기 전까지 요청을 수행하게 된다. 아마 이런식으로 useEffect처럼 작동하는 것이 아닐까 생각한다. 참고로 이런 이유로 mutate를 사용해 캐시를 초기화 하고 싶다면 명시적인 값이나 null을 넣어주어야지 undefined가 반환되면 그대로 유지된다.
